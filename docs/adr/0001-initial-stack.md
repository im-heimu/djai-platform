# ADR 0001: Начальный технологический стек

## Статус

Accepted, in use

## Контекст

DJAI Platform находится на стадии pre-alpha. Проект ведёт один сопровождающий, поэтому важны быстрые итерации и низкая стоимость изменений. Для первого on-prem AI MVP нужен простой и реалистичный путь без лишней операционной и архитектурной сложности. Текущая реализация репозитория уже подтверждает этот выбор: backend на FastAPI, frontend на React/Vite и простой старт через Docker Compose.

## Решение

- Backend: FastAPI
- Frontend: React + Vite
- Начальная архитектура: modular monolith
- Deployment: простой on-prem friendly старт через Docker Compose без преждевременного требования Kubernetes
- Взаимодействие frontend и backend: HTTP API
- Доступ к model: сначала один настроенный LLM endpoint

## Почему так

- FastAPI хорошо подходит для API-first backend и позволяет быстро двигаться на раннем этапе.
- React остаётся практичным вариантом для web-first UI.
- Vite даёт простой старт для минимального frontend без лишней сложности.
- Modular monolith лучше, чем ранний переход к microservices, когда продукт и границы системы ещё не подтверждены.
- Простой deployment снижает порог старта и упрощает первые on-prem установки.
- Узкий стек помогает быстрее проверить основной сценарий платформы.

## Последствия

- запуск будет быстрее
- общая сложность останется ниже
- локальный и ранний on-prem deployment будет проще
- возможная будущая миграция части системы в другие архитектурные формы считается допустимой
- при росте продукта это решение, возможно, придётся пересмотреть
- текущий репозиторий уже следует этому решению, поэтому ADR остаётся актуальным

## Что это решение пока не определяет

- выбор database
- детали реализации auth
- выбор RAG-стека
- выбор observability-стека
- нужен ли queue или broker
- orchestration deployment за пределами первого простого сценария
